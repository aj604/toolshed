# TaskFlow API Reference

The API is an HTTP server defined in `services/api/server.js`. It listens on
`PORT` (default `8080`) and stores tasks in memory (an in-process array), so all
tasks are lost when the process restarts.

## Startup preconditions

The server refuses to start unless both of the following are true:

1. `.taskflow-state.json` exists (created by `make migrate`). Otherwise it
   prints `api: missing .taskflow-state.json — run \`make migrate\` first` and
   exits with code `3`.
2. `DATABASE_URL` is set. Otherwise it prints `api: DATABASE_URL is required`
   and exits with code `1`.

On success it prints `api listening on :<port>`.

## Endpoints

### `GET /health`

Returns `200` with a JSON body reporting status and the current in-memory task
count.

Response:

```json
{ "ok": true, "tasks": 0 }
```

### `POST /tasks`

Creates a task. The request body must be JSON.

Request fields:

| Field      | Type   | Required | Notes                                                        |
| ---------- | ------ | -------- | ----------------------------------------------------------- |
| `title`    | string | yes      | Must be a non-empty string                                  |
| `priority` | string | no       | One of `low`, `med`, `high`; defaults to `med` if omitted   |

Validation is performed by `validateTask` and `normalizePriority` from
`@taskflow/shared`.

Responses:

| Status | Condition                                                  |
| ------ | --------------------------------------------------------- |
| `201`  | Task created; returns the created task as JSON            |
| `400`  | Body is not valid JSON (`bad json`)                       |
| `422`  | Body is valid JSON but fails task validation (`invalid task`) |

A successful response returns the stored task, including its generated `id`:

```json
{ "id": "task_1", "title": "Write docs", "priority": "med" }
```

IDs are generated by `makeId` in `@taskflow/shared`, which produces sequential
values of the form `task_<n>` per process.

### Any other route

Returns `404` with the body `not found`.
