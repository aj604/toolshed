# bloat-scale RED baselines — method

Baseline (RED) runs for the bloat scale-hardening
(`docs/plans/2026-07-07-bloat-scale-hardening-design.md`), run 2026-07-07
against the **pre-edit SKILL.md headless section** (the 0.7.0 text: "you were
handed a manifest and a chunk id. Read the manifest slice you are given…").

The design change under test: headless chunk executors receive their slice
**verbatim in the dispatch prompt** (rendered by `plan-chunks.py
--emit-prompt`); the manifest is orchestration state the executor never
opens. Motivating evidence: career-compass run 28860529836, where every
sweep invocation burned 14–18 turns against `--max-turns 15` and 1–2
permission denials, and 11/35 jobs hit `error_max_turns`.

## Method

- Two sandbox repos under the session scratchpad (`bloat-scale-sandbox/a`:
  one planning-doc sweep chunk + a bait doc outside the slice;
  `bloat-scale-sandbox/b`: a `docs/superpowers` policy chunk), each with a
  real planner manifest; dispatch prompts generated by the **real**
  `--emit-prompt`, not hand-mocked.
- One fresh runner subagent per scenario, given only the sandbox root, the
  skill-files path, and the dispatch verbatim; required to end with a
  complete ordered ACTION LOG + RESULT.
- One fresh **stakeless grader** per scenario, given only the answer key
  (recorded at `scenario-a.md` head), the runner's verbatim output, and the
  SKILL.md path. Graders answer two questions separately: (Q1) did the run
  match the correct action shape; (Q2) does the skill text itself TEACH
  slice-from-the-prompt, or misdirect to a manifest handoff.

## Sequencing note

As with `bloat-rearch-red`, the deterministic scripts (content-addressed
ids, `--emit-prompt`, budgets, `--allow-partial`) were already landed when
RED ran — they carry their own unit-test TDD cycle; the writing-skills Iron
Law gates the *skill text*. The graded criterion is Q2: did the text teach
the contract. It did not, in either scenario.

## Verdicts (details per scenario file)

| Scenario | Run (Q1) | Skill text (Q2) | Core finding |
|---|---|---|---|
| (a) sweep chunk, planning doc | PASS | **MISDIRECTS** | runner complied *despite* the text — the dispatch prompt's scope sentence overrode SKILL.md's "read the manifest slice"; grader: "on a dispatch where [the manifest] is absent, the skill text as written instructs an impossible step" |
| (b) policy chunk | **FAIL** | **MISDIRECTS** | `ls -R` + manifest read caused by the text: "files copied from the manifest verbatim" demands a cross-check the new dispatch makes unnecessary; the enumeration existed to locate the manifest |

Misdirecting sentences catalogued by the graders (all rewritten in GREEN):
the headless paragraph ("you were handed a manifest and a chunk id…"), the
policy sentence ("files copied from the manifest verbatim"), the frontmatter
("handed a manifest slice"), the doc-kinds header ("the manifest hints
these"), and the red flag ("a `files` list that isn't the manifest's
verbatim").
